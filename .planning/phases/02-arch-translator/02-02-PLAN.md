<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| BuildManifest -> emitted profile/installer | Opinion data becomes files in the ISO and pacstrap args at install time. file_asset dst paths and repo sig_levels cross into the target system. |
| variant repo decl -> pacman.conf | sig_level=Never repos downgrade signature trust on the installed system. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08 | Tampering | file_asset dst path traversal (write outside airootfs/target) | mitigate | _sanitize_dst rejects absolute paths and .. escapes; tested with a traversal dst that must raise (ARCH-01 security gate) |
| T-02-09 | Tampering | opinion data shell-injected into generated installer | mitigate | All opinion data serialized to build-manifest.json; installer reads it with jq, never interpolates raw strings (Pitfall 6) |
| T-02-10 | Spoofing | sig_level=Never variant/custom repos in pacman.conf | mitigate | pacman.conf emits an explicit warning comment for every non-Required sig_level repo; keyring_install_before_repos enforced (Pitfall 4) |
| T-02-11 | Elevation | first-run units running as root when user-scope intended | mitigate | first-run units are systemd USER services (etc/systemd/user/), enabled in the target user context, not system scope (research Security Domain) |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs; stdlib + PyYAML + pytest only (Arch-official per research Package Legitimacy Audit) |
</threat_model>
