| pinned image -> built ISO | A stale/compromised base image would propagate into the ISO. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-02-12 | Elevation | --privileged Docker container | accept | Required by mkarchiso (research Pitfall 1); scoped to arch-build-iso.sh only and documented; no privileged step outside the build script |
| T-02-13 | Tampering | stale/compromised pinned Docker image | mitigate | archlinux:base-devel pinned by sha256 digest; Dockerfile comment mandates quarterly digest re-verification (research Security Domain) |
| T-02-14 | Spoofing | sig_level=Never repos reaching the built system | mitigate | northstar/validate surfaces the trust warnings emitted by the generator's pacman.conf comments; documented in README |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No npm/pip/cargo installs; only pacman -Sy archiso inside the pinned Arch image (Arch-official per research Package Legitimacy Audit — all OK) |
