<objective>
Implement translators/debian/ — the second foundation. It mirrors the Arch translator
structurally (same Python-generator + thin-shell pattern, same argv contract, same capability
gate) but wraps Debian's live-build/preseed instead of mkarchiso. Per D-DEB-01.

It imports the shared core from translators/common/ (Plan 04-01): contract loaders, BuildManifest,
firstrun unit rendering. Debian-specific code: capabilities.json (a DIFFERENT, smaller set — no
pacman/mkinitcpio/limine tokens, per D-DEB-03 Findings 3/4), variant.py (sig_level → apt mapping,
the concrete DEB-03 abstraction), and profile.py (emits the live-build config/ tree).

Architecture (04-RESEARCH.md §System Architecture + Patterns 1-4):
- Opinion effectuation goes in config/hooks/live/9000-debateos-apply.hook.chroot (chroot-time,
  no install-time network) — packages via apt, file assets, services, sysctl, kernel params, groups.
  Per resolved open question item 4: apply ALL opinion effectuation in the chroot hook; minimize late_command.
- preseed.cfg (config/includes.installer/) handles d-i automation ONLY (locale, partitioning,
  user account with HASHED password, pkgsel) — never the bulk package install (Pitfall 3).
- custom_repos → config/archives/*.list with sig_level → apt option mapping (Pattern 3 + DEB-03 Finding 1).
- first-run opinions → systemd units via common.firstrun, landing in config/includes.chroot/etc/systemd/user/.

Host gate (04-RESEARCH.md §Host Environment, VERIFIED): lb build fails on this Proxmox host
(loop/devtmpfs); the gate for this plan is profile EMISSION + structural validation. Full lb build
is deferred-to-capable-host (wired in Plan 04-05's debian-build-iso.sh).

Purpose: Prove invariant 1 — a resolved speech becomes a Debian installer config tree (DEB-01).
Output: a complete translators/debian/ package with RED→GREEN pytest coverage.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| opinion data → config/ tree | Curator opinion strings written into preseed.cfg and the chroot hook |
| custom_repos → apt sources | Third-party repo URLs/keyrings declared by opinions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Tampering | file_asset dst in chroot hook | mitigate | _sanitize_dst (copied from arch/profile.py, T-02-08) rejects absolute + ../ traversal; Task 3 test asserts rejection |
| T-04-06 | Tampering | shell injection via opinion data in hook/preseed | mitigate | %%SENTINEL%% replacement only (never str.format with raw opinion data); all opinion payload routed through build-manifest.json (mirrors T-02-01) |
| T-04-07 | Tampering | unsigned custom repo install | mitigate | sig_level Never/OptionalTrustAll → [trusted=yes] + LOUD trust_warning comment in the archive file; Required → signed-by keyring (never silently trust). Task 2 test asserts the warning + mapping |
| T-04-08 | Information Disclosure | preseed password | mitigate | preseed.cfg uses %%HASHED_PASSWORD%% (crypt), never plaintext; Task 3 test asserts the hashed sentinel |
| T-04-09 | Information Disclosure | private pane in config/ tree | accept | Private assets are NOT handled by the translator — they ride private-injection.tar next to the ISO (Phase 3 / build.go), same policy as Arch; out of scope here |
| T-04-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed (pytest + PyYAML already approved; live-build/debootstrap are official Debian apt tools pulled only inside Docker in Plan 04-05) — 04-RESEARCH.md Package Legitimacy Audit |
