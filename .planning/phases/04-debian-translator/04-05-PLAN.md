dual-foundation-check.sh (D-DEB-02) is the centerpiece: resolve examples/dual-foundation ONCE to a
single resolved.json, then run translators/arch/translate AND translators/debian/translate on the SAME
resolved.json, then assert per-foundation equivalence: Arch emits its package set (packages.x86_64 /
build-manifest.json target_packages) + file assets; Debian emits the apt package list
(config/package-lists/*.chroot_install) + preseed + the same file assets. Both must declare every
REQUIRED opinion's tokens (no CapabilityError). The gate runs --skip-iso style (profile emission +
structural checks); full ISO build is the slow gate.

Host gate (04-RESEARCH.md §Host Environment, VERIFIED): lb build fails on this Proxmox host (loop/devtmpfs).
debian-build-iso.sh wraps `docker run --privileged ... lb build` and documents deferred-to-capable-host —
identical policy to scripts/arch-build-iso.sh. debian-validate-iso.sh does the host-runnable structural
validation of the config/ tree.

ownership-model.md (D-COMM-01): distributions own their translators (Ubuntu controls Ubuntu's), curators
own points/speeches, community PRs welcome; how to add a translator = the entrypoint argv contract +
capabilities.json + profile schema. Concise, AI-and-human readable.

Purpose: ship the dual-foundation proof and finalize Phase 4 requirements.
