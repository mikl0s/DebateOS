## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| docker build context -> image | repo + (accidentally) HOME could leak secrets into image layers |
| reusable workflow -> caller forks | cross-repo workflow_call executes on the user's own runner/minutes |
| shared ISO/profile -> public | must be provably secret-free |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-CTX | Information Disclosure | docker build context | mitigate | .dockerignore excludes config dir, *.age, pane.yaml, private-injection.tar; context is repo root not HOME |
| T-03-IMG | Elevation of Privilege | stale archlinux base | mitigate | digest pin identical to Phase 2; quarterly re-verify comment |
| T-03-SECRETFREE | Information Disclosure | shared profile/ISO | mitigate | scripts/secret-free-check.sh greps profile tree; CI gate before publish (PRIV-01) |
| T-03-CIWF | Tampering | GHA workflow | mitigate | least-privilege default (read), explicit upload-artifact; image pulled from public GHCR (no creds); inputs validated as typed |
| T-03-DET | Tampering | reproducibility | mitigate | determinism-test.sh double-run sha256; PAX atime/ctime stripped; gzip -n |
</threat_model>
