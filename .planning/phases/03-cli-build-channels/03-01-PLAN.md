<objective>
Establish the `debateos` CLI foundation: the subcommand dispatch entrypoint, the
shared config-dir resolver, the Runner interface (with recording fake), and the two
read-only subcommands `compose` (resolution preview) and `validate` (CI-friendly
parse + schema-validate + clean-resolve gate). This is the spine every later plan
builds on (CLI-01).

Purpose: Lock the testable contract — `Run(args []string, stdout, stderr io.Writer) int`
with no os.Exit in package logic — so pane/build plans plug in trivially, and so all
external subprocess calls are mockable from day one (D19 TDD requirement).
Output: `cmd/debateos/main.go` + `cli/{config,runner,compose,validate}/` packages with
table-driven tests, and `filippo.io/age` promoted to a direct dependency in go.mod ahead
of the pane plan (single-owner of go.mod).
</objective>
