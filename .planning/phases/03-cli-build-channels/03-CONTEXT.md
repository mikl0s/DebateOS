<decisions>
## Implementation Decisions

### CLI Design (CLI-01/CLI-02, D7)
- stdlib `flag` + small subcommand dispatch in `cli/` (package main at `cmd/debateos/`, logic in `cli/` packages) — module stays minimal-deps (the cobra framework is rejected).
- Speech home: XDG `~/.config/debateos/` (override with `--dir` and `DEBATEOS_DIR` for tests): `speech.yaml` (public panes) + `pane.yaml` (private pane, created 0600).
- `debateos compose` — assemble/edit the speech (add/remove points and opinions from local paths in v1) and print a resolution preview with full explanations (uses resolve.Resolve + Explanation).
- `debateos validate` — parse + schema-validate the speech and all referenced documents; clean-resolve check; non-zero exit on any failure (CI-friendly).
- `debateos build` — resolve (merging private pane locally) → write canonical resolved.json → invoke the Arch translator → invoke the Docker build (channel 1). `--dry-run` stops after emitting the build plan (testable without Docker). `--skip-iso` stops after profile emission (works on hosts that cannot run mkarchiso, like this one).
- `debateos pane` — `get/set/list/backup/restore` for the private pane. All external invocations (docker, git, translator subprocess) go through a Runner interface with a recording fake for tests.
- cmd/resolve-json (Phase 2 seed) is absorbed/kept as the plumbing the CLI reuses; keep it working.

### Private Pane & Secrets (PRIV-01, D16 — design finalized HERE)
- `pane.yaml` lives only in the XDG dir, 0600, never copied into shared artifacts; resolution merges it locally (private opinions overlay public speech).
- Backup: `debateos pane backup` encrypts pane.yaml with age (filippo.io/age Go library — one new dep, audited, tiny) to `pane.yaml.age` and commits/pushes to the user's own private Git repo (user-configured remote); `pane restore` decrypts. Age identity stored at `~/.config/debateos/identity.age` (0600), generated on first use; recovery documented (lose the identity = lose the backup, by design).
- First-boot injection: `debateos build` emits `private-injection.tar` LOCALLY (next to the ISO, never inside it) containing private-pane file assets/secrets; the installer's first-boot unit (generated by the translator since Phase 2) looks for this artifact on mounted removable media at first boot and applies it. Document the flash-time step (copy tar to USB). Shared ISO remains secret-free — verifiable by automated grep/listing test.
- Key-management decision recorded: age X25519 identities, local-only, no escrow, no central service (invariants 4/5/7).

### Build Channels (BLD-01..04, D11)
- ONE image `build/docker/Dockerfile`: digest-pinned archlinux:base-devel + archiso + the debateos CLI binary + translators; entrypoint script: speech dir mounted at /speech, ISO out to /out. Local channel: `docker run debateos:latest` documented + wrapped by `debateos build`.
- Actions channel: `build/actions/build-speech.yml` reusable workflow (workflow_call) + `.github/workflows/build-speech.yml` thin caller in this repo; uses the SAME image; artifact upload of ISO; template-repo usage documented in build/actions/README.md. Live CI execution is a deferred verification item (documented) — local validation = YAML schema sanity + (if available) actionlint.
- Determinism (BLD-03): SOURCE_DATE_EPOCH from resolved-speech sha256 (exists since Phase 2). Automated gate scripts/determinism-test.sh: run resolve+translate TWICE into clean dirs → deterministic tar (--sort=name --mtime=@EPOCH --owner=0 --group=0) → sha256 compare → MUST be identical. Full-ISO determinism uses the same script on a capable host (this host cannot run mkarchiso — documented limitation, same policy as Phase 2).

### TDD (D19)
- Go table-driven tests per subcommand: exit codes, golden stdout, pane perms (assert 0600), dry-run build plan content, fake-Runner command assertions. RED before GREEN.
- Coverage: extend scripts/check-coverage.sh — cli packages ≥85%, resolver packages stay ≥90%.
- Determinism + secret-free-ISO checks are automated scripts, run at wave/phase verification.
