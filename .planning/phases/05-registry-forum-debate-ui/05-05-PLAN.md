## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → OAuth callback | OAuth code + state cross from GitHub redirect (forgery surface) |
| GitHub token → server | Access token obtained then must be discarded (no secrets at rest) |
| registry index → re-index | Re-index source; must be idempotent |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-13 | Spoofing | OAuth state forgery | mitigate | state via crypto/rand stored in cookie; callback rejects mismatch (TestOAuthCallbackValidatesState) |
| T-05-14 | Information Disclosure | OAuth token at rest | mitigate | Token used for user-ID lookup then discarded; never persisted (TestTokenNotPersisted); secrets via env only |
| T-05-15 | Elevation of Privilege | unauthenticated writes | mitigate | Conflict/rating/subscription writes gated on OAuth session; reads public (D13) |
| T-05-16 | Tampering | code-exec / upload boundary | mitigate | Route allowlist asserted; no exec/upload endpoint exists (FORM-05, D13) |
| T-05-17 | Denial of Service | total DB loss | accept→recover | Reindex rebuilds from GitHub-derived registry index; only ephemeral social state lost (FORM-05) |
</threat_model>
