<artifacts_this_phase_produces>
Phase-1 final deliverables:
- **Package `resolver/wasm` (main.go, `//go:build js && wasm`):** registers a `js.Func` (e.g.
  `debateosResolve`) that accepts speech+opinions+hardware JSON/YAML, calls `resolve.Resolve` +
  `resolve.CanonicalJSON`, and returns the canonical JSON string to JS. Consumed by Phase 5 UI.
- **`scripts/wasm-parity-test.sh`:** runs the resolver tests native and under
  `GOOS=js GOARCH=wasm go test -exec="$(go env GOROOT)/lib/wasm/go_js_wasm_exec"`, and asserts the
  canonical-JSON golden output is byte-identical across both targets (RSLV-05). Exit 0 = PARITY OK.
- **`scripts/check-coverage.sh`:** `go test -coverprofile` over `./resolver/...`; fails if total < 90%.
- **`examples/` (4 compositions, CC0):** `omarchy-mini` (real OM-NNN subset), `two-point-clean`,
  `conflicting` (deliberate hard conflict), `hardware-conditional` (hardware-gated). Each is a loadable
  fixture (speech.yaml + opinions.yaml).
- **`examples/examples_test.go`:** loads each example, parses (resolver/parse), resolves (resolver/resolve),
  and asserts the expected end-to-end outcome (clean resolves succeed; conflicting one returns the hard
  conflict; hardware-conditional applies/skips per declared hardware).
</artifacts_this_phase_produces>
