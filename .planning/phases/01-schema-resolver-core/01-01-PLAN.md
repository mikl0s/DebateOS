<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YAML file → parse layer | Untrusted user-authored YAML (opinion/point/speech) crosses into typed Go structs |
| Schema author payloads (custom_repos.sig_level) | Opinion can declare unsigned repos (data only in Phase 1; never executed) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | DoS | resolver/parse YAML decode | mitigate | go.yaml.in/yaml/v3 built-in allowedAliasRatio (billion-laughs protection); no custom parser |
| T-01-02 | Tampering | resolver/parse field binding | mitigate | KnownFields(true) rejects typo'd/injected unknown fields; JSON Schema 2020-12 validation rejects out-of-range values |
| T-01-03 | Elevation | custom_repos sig_level=Never (SR-009) | accept | Phase 1 treats repos as DATA only; sig_level enumerated in schema so trust level is explicit; resolver warning surfaced in 01-04 explanation output, never silently dropped |
| T-01-04 | Tampering | malformed YAML panic | mitigate | parse functions return errors, never panic; no os/file writes from parse layer |
| T-01-SC | Tampering | go get (two modules) | mitigate | Package Legitimacy Audit in RESEARCH.md verified both modules via proxy.golang.org (verdict OK, both approved); no [ASSUMED]/[SUS]/[SLOP] packages — no blocking checkpoint required |
</threat_model>
